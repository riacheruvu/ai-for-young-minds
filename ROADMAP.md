# Roadmap

This project should grow carefully, not constantly.

## Near term

- Publish the first public release on GitHub Pages
- Observe a few independent, family, library, club, or classroom uses
- Fix unclear instructions and accessibility friction
- Add printable versions of the strongest activities
- Map activities to relevant AI4K12 concepts without claiming formal alignment

## Next experiments

Potential additions should earn their place by introducing a new learner action or way of thinking.

Ideas under consideration:

- An **AI Incident Detective** activity about evidence, impact, and missing context
- A child-accessible **model card** for describing capabilities and limits
- A safe **red-team playground** for designing difficult test cases
- An **AI receipts** reflection showing what the learner, tool, and outside sources each contributed

These are possibilities, not promises.

## Maintenance rhythm

- Review links, accessibility, and confusing language each quarter
- Improve or add no more than one substantial activity at a time
- Review safety guidance, audience fit, and AI4K12 connections annually
- Avoid product-specific tutorials unless they teach a durable idea
- Remove outdated examples rather than preserving them for completeness

## Evidence before expansion

Before calling the resource validated—or significantly expanding it—look for:

- Where learners ask their own questions
- Whether they revise an initial claim
- Which instructions require outside explanation
- Which activities hold attention
- What technical depth learners voluntarily open
- What group leaders need to use an activity well

The goal is not to become the largest AI-learning collection. The goal is to remain useful, honest, and worth a learner’s time.
